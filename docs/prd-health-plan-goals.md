# 健康计划页 - 健康管理目标功能 PRD

## 1. 文档说明

### 1.1 背景
本次迭代基于患者档案页中的“健康计划”页面，新增并完善“健康管理目标”相关能力，覆盖：

- 健康目标信息展示
- 健康管理目标概述展示
- 设置目标弹窗配置
- 目标配置结果回显到健康计划页

本期以 MVP 为目标，使用 mock 数据驱动页面与交互，不涉及真实接口联调。

### 1.2 目标
通过本次迭代，支持健管师在患者健康计划中：

- 查看当前已配置的健康目标
- 查看目标的最新数据和目标基线
- 配置和调整重点监测指标
- 填写并回显健康管理目标概述

### 1.3 适用范围
适用页面：

- 患者档案页
- 健康计划模块
- 设置健康管理目标弹窗

不包含：

- 真实 AI 提取接口
- 真实健康档案数据源联动
- 患者端页面

## 2. 页面结构

### 2.1 健康计划页结构
健康计划页当前分为以下板块：

1. 页面标题栏
2. 计划信息与 AI 阶段总结
3. 健康目标
4. 日常打卡
5. 健康任务

### 2.2 健康目标板块结构
健康目标板块从上到下包含：

1. 板块标题：`健康目标`
2. 右侧操作按钮：`设置目标`
3. 健康管理目标概述信息块
4. 已启用指标卡片列表

### 2.3 健康管理目标概述结构
概述区域展示在健康目标板块标题下方，包含：

- 标题：`健康管理目标概述`
- 正文：显示在设置目标弹窗中保存的概述内容

展示规则：

- 若已有概述内容，直接展示保存后的文本
- 当前为轻背景信息块展示，用于承接目标配置说明

### 2.4 健康目标卡片结构
每张卡片代表一个已启用指标，包含：

1. 指标名称
2. 单位
3. 右上角状态标签
4. 左侧字段：`目标基线`
5. 右侧字段：`最新数据`

当前不展示：

- 更新时间
- 溯源入口
- 折叠展开操作

## 3. 健康目标展示规则

### 3.1 卡片生成规则
健康目标卡片基于“设置目标”弹窗中已启用的指标动态生成。

生成规则：

- 仅展示 `enabled = true` 的指标
- 每个启用指标生成一张独立卡片
- 卡片使用小卡片样式，整体尺寸与“日常打卡”卡片统一

### 3.2 卡片字段规则

#### 指标名称
- 显示当前指标的标准名称
- 示例：`收缩压 / 舒张压`、`空腹血糖 (FPG)`、`BMI (身体质量指数)`

#### 单位
- 显示单位值本身
- 不展示“单位”二字
- 示例：`mmHg`、`mmol/L`、`kg/m²`

#### 目标基线
- 标题固定为：`目标基线`
- 数据优先展示用户配置后的 `targetValue`
- 若 `targetValue` 为空，则回退显示模板基线 `baseline`

#### 最新数据
- 标题固定为：`最新数据`
- 展示当前指标的 `currentValue`
- 若为空，则展示：`暂无数据`

### 3.3 状态标签规则
状态标签展示在卡片右上角。

状态文案仅允许以下三种：

- `偏高`
- `偏低`
- `达标`

状态判定基于 mock 规则进行前端判断。

#### 当前已支持的判断指标
- 血压
- 空腹血糖
- 糖化血红蛋白 HbA1c
- LDL-C
- 甘油三酯 TG
- 血清尿酸 SUA
- BMI
- 腰围

#### 状态判定原则
- 低于正常区间下限：显示 `偏低`
- 落在目标区间或正常区间内：显示 `达标`
- 高于正常区间上限：显示 `偏高`

## 4. 设置目标弹窗

### 4.1 弹窗定位
点击健康目标板块右上角 `设置目标` 按钮后，打开“设置健康管理目标”弹窗。

技术约束：

- 强制使用 antd `Modal`
- 弹窗内部交互组件使用 antd 组件体系

### 4.2 弹窗结构
弹窗从上到下分为：

1. 弹窗标题：`设置健康管理目标`
2. 健康管理目标概述输入区
3. 重点监测指标设置说明
4. 左侧树形导航
5. 右侧指标配置区域
6. 底部操作区：`取消`、`保存目标`

## 5. 设置目标弹窗 - 左侧导航

### 5.1 导航结构
左侧采用树形结构，分为一级目录和二级目录。

一级目录：

- 血压管理
- 血糖管理
- 血脂管理
- 尿酸管理
- 体重形态

二级目录：

- 血压管理：收缩压 / 舒张压
- 血糖管理：空腹血糖 (FPG)、糖化血红蛋白 (HbA1c)
- 血脂管理：低密度脂蛋白胆固醇 (LDL-C)、甘油三酯 (TG)
- 尿酸管理：血清尿酸 (SUA)
- 体重形态：BMI (身体质量指数)、腰围

### 5.2 导航样式规则

- 一级与二级目录采用明显缩进错位，体现层级
- 一级目录默认全部展开
- 不提供手动折叠/展开交互
- 左侧菜单宽度已加宽，用于避免二级目录换行
- 菜单整体垂直间距采用紧凑样式

### 5.3 导航交互规则

#### 点击一级目录
- 不触发勾选
- 不切换启用状态
- 当前版本不作为主要操作入口

#### 点击二级目录文本
- 仅切换右侧展示的当前指标
- 不改变该指标的启用状态

#### 点击二级目录复选框
- 仅切换该指标是否启用
- 不影响右侧当前选中项

## 6. 设置目标弹窗 - 右侧指标配置

### 6.1 右侧展示规则
右侧一次只显示一个指标的配置内容，显示逻辑由左侧二级目录决定。

### 6.2 指标配置内容
右侧配置区域包含：

1. 指标名称
2. 指标级别：核心指标 / 辅助指标
3. 启用复选框
4. 现状值
5. 目标基线
6. 指标说明

### 6.3 启用规则

- 右侧顶部保留复选框，用于标识当前指标是否启用
- 该复选框不再控制配置区域展开收起
- 无论是否启用，右侧配置区均直接铺开展示

### 6.4 现状值规则

#### 展示内容
现状值区包含：

- 字段标题：`现状值`
- 提示图标
- `提取档案数据` 按钮
- 输入框
- 数据更新时间
- 简单评估结果

#### 提示文案
hover `现状值` 提示图标时，显示：

`如有数据则自动从健康档案数据提取`

#### 数据更新时间
- 展示在现状值输入框下方
- 含义为：患者健康档案中该指标最新记录的时间
- 不是本次点击“提取档案数据”的操作时间

#### 提取档案数据
- 按钮文案：`提取档案数据`
- 使用 antd 文本按钮
- 点击后进入 antd `loading` 状态
- loading 完成后，回填 mock 的最新指标值到现状值输入框

#### 简单评估结果
根据当前输入值，前端展示轻量状态判断：

- 偏高
- 偏低
- 达标

### 6.5 目标基线规则

#### 展示内容
目标基线区包含：

- 字段标题：`目标基线`
- 提示图标
- 只读输入框

#### 提示文案
hover `目标基线` 提示图标时，显示：

`自动根据客户信息填充正常值范围，可结合实际调整`

#### 编辑规则
当前版本按只读方式展示目标基线，不支持直接编辑。

### 6.6 指标说明
指标说明展示在字段区下方，使用轻文本列表形式展示当前指标说明。

## 7. 健康管理目标概述交互

### 7.1 弹窗内配置
弹窗顶部提供“健康管理目标概述”输入区，包含：

- 标题：`健康管理目标概述`
- 辅助说明：`AI基于患者健康档案数据自动总结，支持自定义`
- 多行输入框

### 7.2 保存规则
- 点击 `保存目标` 后，与指标配置一起保存到页面 mock 状态
- 健康计划页中的“健康管理目标概述”展示区同步更新

## 8. 日常打卡板块

### 8.1 结构
日常打卡板块包含：

1. 标题：`日常打卡`
2. 右侧按钮：`更多打卡记录`
3. 打卡卡片列表

### 8.2 样式规则

- 日常打卡卡片与健康目标卡片使用对齐的小卡片尺寸与弹性布局
- 卡片整体尺寸较之前收小
- 用于与健康目标形成统一的卡片节奏

## 9. 视觉与规范约束

### 9.1 页面级规范

- 页面标题栏复用“报告文档”同类页面框架
- 页面内卡片圆角统一使用 `12px`

### 9.2 组件约束

- 弹窗必须使用 antd `Modal`
- 按钮、输入框、复选框、树形导航等交互组件必须优先使用 antd

### 9.3 板块容器

- 健康目标
- 日常打卡
- 健康任务

以上三个板块均使用统一外层容器包裹，包含：

- 外边框
- 12px 圆角
- 轻背景

### 9.4 标题样式

- 板块标题去掉整块背景
- 标题前保留轻量前导色块
- 标题作为轻强调而非大面积色块标签

## 11. 交互流程

### 11.1 查看健康目标

1. 用户进入患者档案页的健康计划模块
2. 查看健康目标板块
3. 查看概述信息
4. 查看已启用指标卡片

### 11.2 设置目标

1. 点击 `设置目标`
2. 打开设置健康管理目标弹窗
3. 在左侧树形菜单中选择某个二级指标
4. 在右侧查看并调整该指标配置
5. 可勾选或取消勾选二级指标启用状态
6. 可编辑健康管理目标概述
7. 点击 `保存目标`
8. 关闭弹窗并回显到健康计划页

### 11.3 获取最新数据

1. 在右侧指标配置区点击 `提取档案数据`
2. 按钮进入 loading 状态
3. 完成后回填 mock 最新值
4. 当前值评估状态同步更新
