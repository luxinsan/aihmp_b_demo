# AGENT.md

## 项目定位

本项目使用 React 框架构建，目标不是制作简单演示 Demo，而是搭建一个：

**高保真、可运行、可协作开发的前端壳（Production-like Frontend Shell）**

用于：

* 产品方案可视化
* 替代部分静态原型
* 帮助研发准确理解需求
* 支持后端同事提前设计接口
* 提升团队协作效率

---

## 当前开发目标

请将页面开发控制在以下边界内：

### 必须做到

1. 页面视觉效果接近正式系统
2. 主流程交互可走通
3. 使用真实业务结构的 mock 数据展示
4. 补充关键状态反馈

   * loading
   * 空状态
   * 成功反馈
   * 失败反馈
   * 禁用状态
5. 页面之间跳转逻辑清晰

### 暂时不追求

1. 完整生产级工程化
2. 真正后端接口联调
3. 权限体系开发
4. 性能极致优化
5. 复杂状态管理架构
6. 过度组件抽象

---

## 技术栈

* React
* 推荐 Vite
* JavaScript
* CSS Modules / Tailwind 均可

---

## 开发原则

### 原则1：优先保证“像真实产品”

关注：

* 页面完整度
* 信息层级
* UI统一性
* 交互体验

### 原则2：优先保证“研发能接手”

输出结果要让工程同事能够：

* 明确接口字段设计
* 理解页面逻辑
* 快速接入后端能力

### 原则3：不要过度工程化

当前阶段重点是：

**支撑协作，而不是独立上线。**

---

## 数据管理要求

为了避免状态管理混乱：

1. 页面业务数据统一由页面级组件管理
2. 子组件主要负责展示
3. 避免多处重复维护同一份业务数据
4. mock 数据结构尽量贴近真实业务字段

---

## UI设计要求

### 页面至少包含

* 页面标题
* 功能操作区
* 主内容区
* 空状态
* 加载状态
* 分页（如需要）

### 数据展示要求

mock 数据不要过于随意。

请尽量模拟真实业务字段，例如：

* 患者姓名
* 管理状态
* 随访计划
* 风险等级
* 最近监测时间

避免使用无意义测试数据。

---

## 协作目标

最终产出的判断标准：

### 成功标准

如果开发同事看到页面后：

1. 基本不用继续追问需求
2. 能开始定义接口
3. 能快速进入联调开发

说明当前页面已经达到目标。

---

## Codex执行要求

1. 尽量最小范围修改代码
2. 不重构无关模块
3. 保持现有页面风格一致
4. 优先完成当前任务，而非额外优化
5. 输出修改说明

---

## 一句话总结

**当前项目目标是：构建让研发团队可以直接开工的高保真前端壳，而不是独立完成生产系统。**
